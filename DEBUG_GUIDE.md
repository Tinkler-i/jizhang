# 账单导入功能 - 调试指南

## 快速定位错误

### 1. 查看浏览器控制台（前端日志）

1. 打开浏览器开发者工具（F12）
2. 切换到"Console"标签
3. 上传图片并查看日志输出

**查找关键信息：**
```
开始识别账单，图片大小: xxx 字节
账单类型: xxx
API响应: {...}
识别成功，记录数: x
```

**如果出错，会显示：**
```
识别请求错误: xxx
错误详情: {...}
识别失败: xxx
```

### 2. 查看后端日志

**方法一：直接在启动终端查看**
- 如果使用 `mvn spring-boot:run` 启动，日志会直接输出到控制台
- 查找以下关键日志：

```
用户 X 提交账单识别请求，账单类型: xxx, 图片大小: xxx 字节
开始调用讯飞API识别图像，账单类型: xxx
讯飞API配置验证通过
开始调用讯飞API...
讯飞API调用成功，响应长度: xxx 字符
识别成功，识别到 x 条记录
```

**方法二：查看日志文件**
```bash
# 如果配置了日志文件
tail -f logs/jizhang.log
```

**方法三：使用提供的脚本**
```bash
# Windows
view-logs.bat

# Linux/Mac
tail -f logs/spring.log
```

### 3. 常见错误和解决方案

#### 错误1：讯飞API未配置
**日志显示：**
```
识别失败: 图像识别失败: 讯飞API未配置 - 请在application.yml中配置app-id
```

**解决方案：**
编辑 `src/main/resources/application.yml`，配置真实的API凭证：
```yaml
xunfei:
  api:
    app-id: 你的真实APP_ID
    api-key: 你的真实API_KEY
    api-secret: 你的真实API_SECRET
```

#### 错误2：API调用超时
**日志显示：**
```
API调用超时，未在 30000 毫秒内收到响应
识别失败: 图像识别失败: API调用超时（超过30秒）
```

**可能原因：**
- 网络连接问题
- 图片过大
- 讯飞服务响应慢

**解决方案：**
1. 检查网络连接
2. 使用较小的图片（< 2MB）
3. 增加超时时间（修改 application.yml）

#### 错误3：WebSocket连接错误
**日志显示：**
```
WebSocket连接错误: xxx
识别失败: API调用失败: xxx
```

**可能原因：**
- API凭证错误
- 签名生成错误
- 网络防火墙阻止WebSocket连接

**解决方案：**
1. 验证API凭证是否正确
2. 检查网络防火墙设置
3. 查看详细错误信息确定具体问题

#### 错误4：未能识别到有效信息
**日志显示：**
```
识别失败: 未能从图像中识别到有效的账单信息
```

**可能原因：**
- 图片不清晰
- 图片不是账单截图
- AI无法识别图片中的文字

**解决方案：**
1. 使用清晰的账单截图
2. 确保图片包含明确的金额、日期等信息
3. 尝试不同的图片

#### 错误5：用户未登录
**日志显示：**
```
识别请求失败：用户未登录
```

**解决方案：**
1. 刷新页面重新登录
2. 检查Session是否过期

### 4. 获取完整错误信息的步骤

**步骤1：开启调试日志**

编辑 `application.yml`，设置日志级别为DEBUG：
```yaml
logging:
  level:
    com.billmanager.jizhang: DEBUG
```

**步骤2：重启应用**
```bash
mvn clean spring-boot:run
```

**步骤3：打开浏览器控制台**
- 按 F12 打开开发者工具
- 切换到 Console 标签
- 保持打开状态

**步骤4：上传图片**
- 选择一张账单图片
- 点击"识别账单"

**步骤5：收集信息**

**从浏览器控制台复制：**
- 所有以"开始识别"、"API响应"、"识别失败"开头的日志
- 错误详情对象的完整内容

**从后端日志复制：**
- 所有包含"账单识别"、"讯飞API"的日志行
- 完整的错误堆栈信息

### 5. 调试检查清单

使用前端时，按以下顺序检查：

- [ ] 应用是否正常启动（访问 http://localhost:8080/jizhang/ 能看到登录页）
- [ ] 是否成功登录
- [ ] 能否访问账单导入页面
- [ ] 浏览器控制台是否有错误
- [ ] 上传图片后前端是否显示错误消息
- [ ] 后端控制台/日志是否有错误输出

配置检查：

- [ ] application.yml 中 xunfei.api 配置是否填写
- [ ] app-id 不是 "YOUR_APP_ID"
- [ ] api-key 不是 "YOUR_API_KEY"  
- [ ] api-secret 不是 "YOUR_API_SECRET"
- [ ] 网络能否访问 spark-api.xf-yun.com

### 7. 测试API配置是否正确

运行以下测试命令验证配置：

```bash
# 编译并运行
mvn clean package -DskipTests
java -jar target/jizhang-1.0.0.jar

# 查看启动日志，确认没有配置错误
```

启动成功后，应该能看到类似：
```
Started JizhangApplication in x.xxx seconds
```

### 8. 网络调试

查看浏览器Network标签：
1. 打开开发者工具（F12）
2. 切换到 Network 标签
3. 上传图片
4. 查找 `/api/bill-import/recognize` 请求
5. 查看请求和响应详情

**正常响应示例：**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "records": [...]
  }
}
```

**错误响应示例：**
```json
{
  "code": 500,
  "msg": "识别失败: 具体错误信息",
  "data": null
}
```
