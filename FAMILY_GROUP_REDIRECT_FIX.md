# 家庭组管理 - 用户退出后重定向修复

## 问题描述

用户从家庭组中删除自己后，仍然能在家庭组管理页面看到原来的家庭组信息。正确的行为应该是：
- 检测用户不在任何家庭组中
- 自动导向到"加入/创建家庭"界面
- 允许用户创建新家庭或加入现有家庭

### 2. 后端API新增

**文件**: `src/main/java/com/billmanager/jizhang/controller/FamilyGroupController.java`

#### 2.1 创建家庭组端点
```
POST /api/family-groups/create
```

**请求参数**:
```json
{
  "name": "家庭名称"
}
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "name": "王家",
    "code": "ABC123",
    "creatorId": 123,
    "description": "",
    "createTime": "2026-01-13T22:25:00"
  },
  "message": "家庭组创建成功"
}
```

**验证规则**:
1. ✅ 用户必须认证
2. ✅ 家庭组名称不能为空
3. ✅ 用户不能同时属于多个家庭组（检查是否已在其他家庭组中）
4. ✅ 创建者自动成为新家庭组的管理员

**错误处理**:
- 400: 家庭组名称为空或用户已加入家庭组
- 401: 用户未认证
- 500: 服务器错误

### 3. 前后端交互流程

```
用户访问家庭组管理页面
    ↓
onMounted 执行
    ↓
1. 调用 getCurrentUser() 获取当前用户ID
    ↓
2. 调用 loadFamilyGroup()
    │
    ├─ 请求 GET /api/family-groups/current
    │
    ├─ 成功 (200):
    │   ├─ 设置 familyGroup
    │   ├─ 加载成员列表
    │   └─ 切换到 'info' 标签
    │
    ├─ 404 (未加入家庭):
    │   ├─ 清空 familyGroup
    │   ├─ 清空 members
    │   └─ 切换到 'join' 标签 ⭐
    │
    └─ 异常:
        ├─ 清空 familyGroup
        └─ 切换到 'join' 标签 ⭐
    ↓
用户看到"加入或创建家庭"界面 ⭐
    │
    ├─ 选择创建新家庭
    │   └─ 调用 createFamily()
    │       └─ POST /api/family-groups/create
    │           └─ 重新加载家庭组信息
    │
    └─ 选择加入现有家庭
        └─ 调用 joinFamily()
            └─ POST /api/family-members/join
                └─ 重新加载家庭组信息
```

## 使用场景

### 场景1: 用户首次使用
1. 打开应用 → 进入家庭组管理
2. 检测到未加入家庭 → 显示"加入或创建家庭"界面
3. 用户可选择：
   - 创建新家庭（输入家庭名称）
   - 加入现有家庭（输入编号）

### 场景2: 用户从家庭组删除自己
1. 用户在成员列表中点击自己的操作按钮
2. 选择"退出家庭"并确认
3. 后台删除用户在该家庭组中的成员记录
4. 刷新页面时：
   - 调用 `/api/family-groups/current` 返回 404
   - 前端自动切换到"加入或创建家庭"界面 ⭐
   - 用户无法看到原来的家庭组信息

### 场景3: 多家庭组环境（未来支持）
- 目前逻辑只支持用户属于一个家庭组
- 创建API会检查并阻止用户创建多个家庭组
- 返回错误信息："您已经加入了一个家庭组，无法同时加入多个家庭组"
